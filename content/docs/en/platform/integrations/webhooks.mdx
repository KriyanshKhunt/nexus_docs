---
title: Webhook Channel
description: Send compiled notification payloads to your own HTTP endpoints.
---

The **Webhook** channel POSTs structured JSON to a URL you configure — ideal for internal microservices, Zapier, or custom automation.

## Setup

1. Add **Custom Webhook** in Integrations (`webhook_custom`)
2. Set URL, auth method, and optional signing secret
3. Add a **Webhook** channel node to your workflow

## Outbound payload

```json
{
  "event": "notification.delivered",
  "workflow": { "id": "…", "name": "order.shipped" },
  "subscriber": { "id": "…", "externalId": "user_42" },
  "channel": "WEBHOOK",
  "data": { "orderId": "ORD-9912" },
  "compiled": { "title": "…", "body": "…" }
}
```

## Authentication

| Method | Headers |
|--------|---------|
| None | — |
| Bearer | `Authorization: Bearer …` |
| Basic | `Authorization: Basic …` |
| HMAC | `x-nexus-signature`, `x-nexus-timestamp` |

Verify HMAC on your server using the configured signing secret.

## Reliability

Failed webhook deliveries retry via BullMQ. Responses are stored in audit log `responseMeta`.
