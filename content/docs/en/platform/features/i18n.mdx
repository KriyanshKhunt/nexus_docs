Ship to global users without duplicating workflows.

## Template structure

Store locale variants in `contentTemplates`:

```json
{
  "en": { "email_subject": "Your order shipped", "sms_body": "Shipped!" },
  "es": { "email_subject": "Tu pedido fue enviado", "sms_body": "¡Enviado!" },
  "fr": { "email_subject": "Votre commande est expédiée", "sms_body": "Expédié !" }
}
```

## Locale resolution order

1. Per-trigger `_locale` override (from recipient on trigger)
2. Subscriber `locale` field
3. Fallback to `en`

## Set locale on trigger

```ts
recipients: [{ externalId: 'user_1', locale: 'es' }]
```

## Supported locales

`en`, `es`, `fr`, `de`, `pt`, `it`, `ja`, `ko`, `zh`, `ar`, `hi`, `nl`, `pl` — validated as BCP 47 tags.

## Template editor

Use **locale tabs** in the dashboard template studio to add/remove languages. The device emulator previews each locale.

## AI generation

AI content generation accepts a `locale` parameter to produce translated copy in one step.
