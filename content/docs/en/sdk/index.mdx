# Nexus Signal SDKs

Official packages for triggering workflows from your backend and rendering in-app notifications in the browser.

<Cards>
  <Card title="Node.js SDK" href="/docs/sdk/node" description="Triggers, schedules, HMAC — server-side only." />
  <Card title="React SDK" href="/docs/sdk/react" description="Bell, inbox, preferences — browser UI." />
  <Card title="API Reference" href="/docs/api" description="REST endpoints if you prefer raw HTTP." />

## Integration flow

```mermaid
flowchart LR
  BE[Your backend] -->|sk_* trigger| API[Nexus API]
  BE -->|HMAC| FE[React app]
  API --> W[Worker]
  W -->|WebSocket| FE
  W --> P[Your providers]
```

1. Backend triggers workflows with the **secret key** (`sk_*`)
2. Backend generates **HMAC** for the logged-in user
3. Frontend wraps the app in `NexusProvider` with **public key + HMAC**
4. In-app notifications stream to `NotificationCenterBell`

## Quick trigger example

```ts
import { NexusClient } from '@nexus-signal/node';

const nexus = new NexusClient({
  secretKey: process.env.NEXUS_SECRET_KEY!,
  baseUrl: 'https://api.nexussignal.dev',
});

await nexus.workflows.trigger({
  workflowName: 'order.shipped',
  recipients: [{ externalId: 'user_42', email: 'alex@acme.io' }],
  data: { trackingNumber: '1Z999AA10123456784' },
});
```

```bash
npm install @nexus-signal/react socket.io-client
```

  </Tab>
  <Tab value="pnpm">

```bash
pnpm add @nexus-signal/node
pnpm add @nexus-signal/react socket.io-client
```

  </Tab>
  <Tab value="yarn">

```bash
yarn add @nexus-signal/node
yarn add @nexus-signal/react socket.io-client
```

  </Tab>
</Tabs>

<Callout type="info">
REST-only integration? Skip the React SDK and use [API Reference](/docs/api) with your HTTP client.
</Callout>

<Callout type="warn">
Never expose `sk_*` secret keys in browser code. Use `pk_*` public keys for SDK routes only.
</Callout>
