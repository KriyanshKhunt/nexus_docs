---
title: Delivery Windows
description: Timezone-aware delivery — only send between allowed hours in the subscriber's local time.
---

**Delivery Window** nodes ensure notifications never arrive at inappropriate hours (e.g. 9 AM–9 PM in the user's timezone).

## Canvas setup

Add a **Delivery Window** node with:

- `windowStart` — `"09:00"`
- `windowEnd` — `"21:00"`
- `urgent` — bypass window for critical alerts (optional)

## Subscriber timezone

Set on the subscriber profile or pass per trigger:

```ts
recipients: [{ externalId: 'user_1', timezone: 'Europe/London' }]
```

Uses IANA timezone identifiers. Falls back to UTC if unset.

## Behavior

| Situation | Action |
|-----------|--------|
| Inside window | Continue immediately |
| Outside window | Hold in queue; BullMQ delayed job resumes at next window open |
| `urgent: true` | Deliver immediately regardless |

## Use cases

- Marketing emails — business hours only
- Push notifications — avoid sleep hours
- SMS — compliance-friendly quiet hours
