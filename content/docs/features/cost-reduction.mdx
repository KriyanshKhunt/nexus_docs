Nexus Signal includes several features designed to **reduce unnecessary provider spend** without sacrificing deliverability.

## Presence suppression

When a subscriber has an **active WebSocket** connection (they're using your app right now), Nexus can skip expensive external channels and deliver **in-app only**.

### Default behavior

For channel nodes, these channels are suppressed when the user is online:

- Web Push
- Mobile Push
- SMS
- WhatsApp

**In-app** always delivers over WebSocket. Enable `suppressWhenOnline` on the channel node to control this explicitly.

### How it works

1. React SDK maintains a WebSocket with heartbeat
2. Redis tracks `presence:sub:{subscriberId}` (45s TTL)
3. Worker checks presence before dispatching external channels
4. Skipped deliveries log `SKIPPED_BY_PREFERENCE` with lifecycle step `PRESENCE_SUPPRESSION`

**Result:** No redundant $0.05 SMS when the user is already staring at your app.

## Sandbox mode (Development)

In the **Development** environment, outbound provider calls are replaced with a **virtual simulator**. You see push, SMS, and email previews in the dashboard without API spend.

See [Sandbox mode](/docs/features/sandbox).

## Weighted provider routing

Run multiple provider accounts per channel and split traffic by percentage:

```json
{ "sendgrid": 70, "resend": 30 }
```

Configure in **Integrations → Weight sliders**. The worker rolls 1–100 and picks a provider — useful for gradual migrations and load spreading.

## Circuit breaker failover

When a provider fails repeatedly (5 errors in 60s), the circuit opens and traffic routes to healthy alternatives or workflow failover branches — avoiding wasted retry spend on a dead endpoint.

## Combined impact

Teams typically reduce spend by:

1. Suppressing redundant push/SMS for active users
2. Testing in sandbox before production sends
3. Routing away from failing providers automatically
