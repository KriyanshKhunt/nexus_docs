---
title: Failover
description: Automatic channel fallback when primary delivery fails or goes unread.
---

The **Failover** node guarantees critical alerts reach users even when the primary channel fails.

## Configuration

| Field | Description |
|-------|-------------|
| `primaryChannel` | First attempt (e.g. `MOBILE_PUSH`) |
| `fallbackChannel` | Backup (e.g. `SMS`) |
| `timeoutSeconds` | Wait for read ack before fallback (10–3600, default 120) |

## Flow

1. Dispatch primary channel → status `DISPATCHED`
2. Start Redis countdown watch for read acknowledgment
3. If user opens notification (WebSocket/SDK read event) → done
4. If timeout expires without read → dispatch fallback channel
5. Immediate fallback if primary fails or circuit breaker is open

## Example

> Send push notification. If unread after 120 seconds, send SMS.

Perfect for OTP, security alerts, and appointment reminders.

## Chaos mode testing

Enable **Chaos Mode** in Development integrations to simulate provider failures and watch failover execute live in audit logs.
