---
title: 配信ウィンドウ
description: タイムゾーン対応配信 — サブスクライバーのローカル時間で許可された時間帯のみ送信。
---

**配信ウィンドウ**ノードは、通知が不適切な時間帯（例: ユーザーのタイムゾーンで午前9時〜午後9時）に届かないようにします。

## キャンバス設定

以下で**配信ウィンドウ**ノードを追加します:

- `windowStart` — `"09:00"`
- `windowEnd` — `"21:00"`
- `urgent` — 緊急アラートのウィンドウをバイパス（オプション）

## サブスクライバーのタイムゾーン

サブスクライバープロファイルで設定するか、トリガーごとに渡します:

```ts
recipients: [{ externalId: 'user_1', timezone: 'Asia/Tokyo' }]
```

IANA タイムゾーン識別子を使用します。未設定の場合は UTC にフォールバックします。

## 動作

| 状況 | アクション |
|-----|---------|
| ウィンドウ内 | 即座に続行 |
| ウィンドウ外 | キューで保留；BullMQ 遅延ジョブが次のウィンドウ開始時に再開 |
| `urgent: true` | 関係なく即座に配信 |

## ユースケース

- マーケティングメール — 営業時間のみ
- プッシュ通知 — 睡眠時間を避ける
- SMS — コンプライアンス対応のサイレント時間
