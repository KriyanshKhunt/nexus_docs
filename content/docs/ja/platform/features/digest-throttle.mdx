## ダイジェストノード

スライディングウィンドウ（デフォルト **30分**）後に高頻度イベントを**1つのサマリー通知**に折りたたみます。

### 動作

1. ダイジェストノードにヒットしたイベントは、サブスクライバー/ワークフロー/ノードごとに Redis リストに保存
2. ステータスを `QUEUED_IN_DIGEST` に設定
3. ウィンドウが満了すると、ワーカーがバンドルされた `triggerData.digest` アイテムでサマリーテンプレートをコンパイル
4. 単一通知をディスパッチ

**ユースケース:** 12個の別々のプッシュではなく「12件の新しいコメントがあります」。

## スロットルノード

時間ウィンドウ内に通過する通知数をレート制限します。

| 設定 | 例 |
|------|--|
| `maxCount` | 3 |
| `windowSeconds` | 60 |
| `divertToDigest` | true — オーバーフローはドロップではなくダイジェストへ |

**ユースケース:** 一括インポートや DDoS 的なイベントスパイク時のアラートストームを防止。
