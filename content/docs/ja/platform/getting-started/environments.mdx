すべての Nexus ワークスペースには**3つの環境**が含まれます。それぞれがワークフロー、サブスクライバー、テンプレート、スケジュール、トピックのための完全に独立したスコープです。

## 環境マトリクス

| | 開発 | ステージング | 本番 |
|---|------|------------|------|
| **公開キー** | `pk_dev_*` | `pk_stg_*` | `pk_prd_*` |
| **シークレットキー** | `sk_dev_*` | `sk_stg_*` | `sk_prd_*` |
| **サンドボックスモード** | あり — シミュレート配信 | なし | なし |
| **カオスモード** | オプション — フェイルオーバーテスト | なし | なし |
| **実際のプロバイダー** | サンドボックスパスを無効にした場合のみ | あり | あり |

## サンドボックス（開発環境のみ）

**開発**環境では、アウトバウンドチャンネル（アプリ内を除く）は実際のキャリア API の代わりに**仮想サンドボックス**を通じてルーティングされます。ペイロードはダッシュボードシミュレーターへ WebSocket でストリーミングされます — **プロバイダーのクレジットを使わずに**複雑なワークフローをテストできます。

詳細は[サンドボックスモード](/docs/platform/features/sandbox)をご覧ください。

## 環境の切り替え

ダッシュボードのトップバーにある環境ドロップダウンを使用してください。すべての API 呼び出しはその環境に対応するシークレットキーを使用する必要があります。

<Callout type="warn">
git にコミットされたローカルの `.env` ファイルに本番キーを使用しないでください。ローカル作業には開発キーを使用してください。
</Callout>

## サブスクライバーは環境ごと

開発環境での `externalId: user_123` のサブスクライバーは、本番環境の同じ ID とは**異なるレコード**です。統合を昇格させるときはサブスクライバーを同期またはマイグレーションしてください。
