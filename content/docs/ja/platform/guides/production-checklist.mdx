ライブトラフィックを本番環境にルーティングする前にこのチェックリストを使用してください。

## キーと環境

- [ ] 本番シークレットキー（`sk_prd_`）はシークレットマネージャーに保存済み — git には絶対に入れない
- [ ] ステージングは `sk_stg_` と実際のプロバイダーキーでテスト済み
- [ ] 公開キー（`pk_prd_`）はフロントエンド環境変数のみに設定

## プロバイダー

- [ ] 必要なすべてのプロバイダーが接続済みで**接続テスト**に合格
- [ ] レイヤーごとに複数のプロバイダーを使用する場合はウェイト分割を設定
- [ ] [コスト分析](/docs/platform/features/cost-analytics)のためのプロバイダー単価コストを設定
- [ ] オプションの月次予算を設定し、アラートメールを確認済み

## ワークフロー

- [ ] テンプレートは **Production Active（本番アクティブ）** に昇格済み（ドラフトではない）
- [ ] ワークフローが**アクティブ**としてマーク済み
- [ ] 開発カオスモードでフェイルオーバーパスをテスト済み
- [ ] ユーザー向けマーケティング送信の配信ウィンドウを設定
- [ ] トピックを割り当て済みで設定マトリクスをテスト済み

## SDK 統合

- [ ] HMAC はセッションごとにサーバーサイドで生成
- [ ] Node SDK サーキットブレーカー / リトライ処理を実装
- [ ] サブスクライバーの `externalId` は認証ユーザー ID と一致

## コンプライアンスと運用

- [ ] GDPR 削除プロセスを文書化済み（サブスクライバー削除 API）
- [ ] チームロールを割り当て済み（オーナー、開発者、マーケター）
- [ ] 管理者アカウントで MFA が有効
- [ ] 監査ログの保持期間とモニタリング計画を定義済み

## 請求

- [ ] プランティアが予想ボリュームと一致
- [ ] 支出上限の設定を完了
- [ ] 管理者の Stripe 請求ポータルへのアクセスを確認

## 本番稼働（Go live）

1. ダッシュボード環境を **Production（本番）** に切り替え
2. `sk_prd_` シークレット付きでバックエンドをデプロイ
3. 社内チームにカナリアトリガーを送信
4. 最初の1時間、**監査ログ**と**概要**を監視

<Callout type="warn">
開発サンドボックスは本番環境では**動作しません**。すべてのトリガーは実際のプロバイダーを通じて送信され、プロバイダー費用が発生します。
</Callout>
