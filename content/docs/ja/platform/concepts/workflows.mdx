**ワークフロー**は、ビジュアルキャンバスから**ステップツリー**を持つ名前付き自動化（`order.shipped`、`user.signup`）です。

## トリガー

すべてのワークフローはイベント名前空間を定義する**トリガー**ノードから始まります。SDK 呼び出しはこの識別子に一致する `workflowName` を使用します。

## ノードタイプ

| ノード | 目的 |
|------|-----|
| **チャンネル** | メール、SMS、プッシュ、アプリ内、WhatsApp、Slack、Discord、Teams、Webhook で送信 |
| **遅延** | 次のステップまで固定時間待機 |
| **指定時刻まで待機** | トリガーペイロードからのタイムスタンプまで待機（例: 予約24時間前） |
| **配信ウィンドウ** | サブスクライバーのローカルタイムゾーンの指定時間帯のみ配信 |
| **ダイジェスト** | 高頻度イベントを1つのサマリーにまとめる（デフォルト30分ウィンドウ） |
| **スロットル** | サブスクライバーごとのアラートレート制限；オプションでダイジェストに転送 |
| **フェイルオーバー** | プライマリチャンネル → 既読を待つ → フォールバックチャンネル |
| **A/B 分割** | エンゲージメント追跡付きのパーセンテージトラフィック分割 |
| **スマートタイミング** | チャンネルノードでの切り替え — サブスクライバーのピーク時間に遅延 |

## チャンネルノードオプション

**チャンネル**ノードで以下を有効にできます:

- **`suppressWhenOnline`** — ユーザーがアクティブな WebSocket を持っている場合は外部チャンネルをスキップ（[コスト削減](/docs/platform/features/cost-reduction)を参照）
- **`smartTiming`** — AI 送信時刻最適化（[スマート送信時刻](/docs/platform/features/smart-send-time)を参照）

## テンプレート

各ワークフローにはバージョン管理された**テンプレート**があります（ドラフト → 本番アクティブ）。コンテンツは Handlebars 変数をサポートします:

```html
<p>{{ subscriber.externalId }} 様、注文 {{ data.orderId }} が発送されました。</p>
```

多言語テンプレートはロケールキーを使用します — [i18n](/docs/platform/features/i18n)をご覧ください。

## ワークフロー ↔ トピック

ワークフローを[トピック](/docs/platform/features/topics)に割り当てることで、サブスクライバーが個別のイベント名ではなくカテゴリで設定を管理できます。
