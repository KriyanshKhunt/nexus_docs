# Nexus Signal API

**ベース URL:** `https://api.nexussignal.dev/v1`

トリガー、アプリ内受信箱、スケジュール、サブスクライバー管理、分析へのプログラムアクセスを提供します。すべての取り込みエンドポイントは **202 Accepted** を返します — 配信は非同期です。

## 認証モード

| 対象 | 認証 | エンドポイント |
|----------|------|-----------|
| サーバーアプリ | `Authorization: Bearer sk_*` | `/workflows/trigger`, `/schedules/sdk` |
| ブラウザ SDK | `x-nexus-public-key: pk_*` | `/sdk/*` |
| ダッシュボード | JWT セッションクッキー / Bearer | `/workflows/dashboard`, `/analytics`, `/subscribers` など |

## レスポンス規約

- トリガーは **202 Accepted** を返します（非同期処理）
- エラーは `{ error: string, code?: string, details?: object }` を返します
- レート制限はプランティアごとに適用されます

## ヘルス & テレメトリ

<Tabs items={['ヘルス', 'テレメトリ']}>
  <Tab value="ヘルス">

```http
GET /health
GET /health/deep
```

  </Tab>
  <Tab value="テレメトリ">

```http
GET /v1/telemetry/click/:trackingId
POST /v1/telemetry/callbacks/:provider
```

クリックトラッキングはユーザーをリダイレクトし、エンゲージメントを記録します。キャリアコールバックは配信ステータスを更新します。

  </Tab>
</Tabs>

<Callout type="warn">
ブラウザコードに `sk_*` シークレットキーを公開しないでください。SDK ルートには `pk_*` 公開キーのみを使用してください。
</Callout>

<Callout type="info">
型付きの統合をお好みですか？[Node SDK](/docs/sdk/node) と [React SDK](/docs/sdk/react) をご覧ください。
</Callout>
