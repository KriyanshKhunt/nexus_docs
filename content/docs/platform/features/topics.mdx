**Topics** let users opt in/out of **categories** instead of individual workflow event names.

## Model

| Field | Description |
|-------|-------------|
| `name` | Machine key (e.g. `marketing`) |
| `displayName` | UI label |
| `defaultStatus` | `OPTED_IN`, `OPTED_OUT`, or `REQUIRED` |
| `sortOrder` | Preference matrix ordering |

Max **50 topics** per environment.

## Assign workflows

Link a workflow to a topic via `topicId`. Preference evaluation checks `topic:{name}` keys on the subscriber profile.

## Dashboard

Manage topics under **Settings → Topics**. Assign workflows from the workflow editor dropdown.

## SDK

```http
GET /v1/sdk/topics
x-nexus-public-key: pk_prd_…
```

Also included in `GET /v1/sdk/notification-events` for the preference matrix.

## React SDK

`<SubscriberPreferenceMatrix groupByTopic />` renders topics as rows with channel checkboxes.
