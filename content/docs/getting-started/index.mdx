Welcome to Nexus Signal. This section walks you from zero to a working trigger in three steps: **workspace → providers → SDK trigger**.

## Prerequisites

- A Nexus Signal account ([sign up free](https://app.nexussignal.dev))
- Node.js 18+ for the server SDK
- At least one provider account (SendGrid, Resend, Twilio, etc.)

## Recommended path

1. [Quickstart](/docs/getting-started/quickstart) — first workflow trigger in under 5 minutes
2. [Environments](/docs/getting-started/environments) — Development, Staging, Production keys
3. [Authentication](/docs/getting-started/authentication) — secret keys, public keys, and HMAC for WebSockets

## How delivery works (30-second overview)

```
Your app  →  POST /v1/workflows/trigger  →  202 Accepted
                    ↓
              BullMQ worker  →  workflow steps  →  your providers
                    ↓
         In-app: WebSocket  |  Email/SMS/Push: SendGrid, Twilio, FCM…
```

Every delivery is logged with full lifecycle states for debugging and analytics.
