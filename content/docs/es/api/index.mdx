# API de Nexus Signal

**URL base:** `https://api.nexussignal.dev/v1`

Acceso programático a triggers, inbox in-app, schedules, gestión de suscriptores y analítica. Los endpoints de ingestión devuelven **202 Accepted** — la entrega es asíncrona.

## Modos de autenticación

| Audiencia | Auth | Endpoints |
|-----------|------|-----------|
| Apps servidor | `Authorization: Bearer sk_*` | `/workflows/trigger`, `/schedules/sdk` |
| SDK browser | `x-nexus-public-key: pk_*` | `/sdk/*` |
| Dashboard | JWT session / Bearer | `/workflows/dashboard`, `/analytics`, `/subscribers`, etc. |

## Convenciones de respuesta

- Los triggers devuelven **202 Accepted** (procesamiento async)
- Errores: `{ error: string, code?: string, details?: object }`
- Rate limits según plan
