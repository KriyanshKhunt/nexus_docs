Nexus ships two npm packages:

| Package | Purpose |
|---------|---------|
| [`@nexus-signal/node`](/docs/sdks/node) | Server-side workflow triggers, schedules, HMAC |
| [`@nexus-signal/react`](/docs/sdks/react) | In-app bell, inbox, preference matrix |

## Typical integration

```
┌──────────────┐   trigger + HMAC    ┌──────────────┐
│ Your backend │ ──────────────────► │ Nexus API    │
└──────┬───────┘                     └──────┬───────┘
       │ hmacSignature                      │ WebSocket
       ▼                                    ▼
┌──────────────┐                     ┌──────────────┐
│ Your React   │ ◄── notifications ──│ Worker       │
│ app (SDK)    │                     └──────────────┘
└──────────────┘
```

1. Backend triggers workflows with secret key
2. Backend generates HMAC for logged-in user
3. Frontend wraps app in `NexusProvider` with public key + HMAC
4. Real-time in-app notifications appear in `NotificationCenterBell`

## Requirements

| SDK | Requires |
|-----|----------|
| Node | Node.js 18+ |
| React | React 18/19, `socket.io-client` ^4.8 |

## Install

```bash
npm install @nexus-signal/node
npm install @nexus-signal/react socket.io-client
```
