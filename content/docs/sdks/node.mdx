Server-side SDK for triggering workflows from your API routes, workers, or scripts.

## Install

```bash
npm install @nexus-signal/node
```

## Initialize

```ts
import { NexusClient } from '@nexus-signal/node';

const nexus = new NexusClient({
  secretKey: process.env.NEXUS_SECRET_KEY!, // sk_prd_…
  baseUrl: 'https://api.nexussignal.dev',
  timeout: 5000,
});
```

## Trigger a workflow

```ts
const result = await nexus.workflows.trigger({
  workflowName: 'billing.invoice_overdue',
  recipients: [
    {
      externalId: 'cust_8824',
      email: 'billing@enterprise.com',
      phoneNumber: '+15550198234',
      timezone: 'America/New_York',
      locale: 'en',
      slackWebhookUrl: 'https://hooks.slack.com/services/…',
    },
  ],
  data: {
    invoiceId: 'INV-2026-0049',
    amountDue: '$1,250.00',
  },
});

if (!result.ok) {
  console.warn(result.error, result.queued ? '(buffered for retry)' : '');
}
```

### Recipient fields

| Field | Type | Description |
|-------|------|-------------|
| `externalId` | string | Your user ID (required unless `subscriberId`) |
| `subscriberId` | UUID | Nexus subscriber ID |
| `email` | string | For email channel |
| `phoneNumber` | E.164 | For SMS/WhatsApp |
| `timezone` | IANA | Updates subscriber timezone |
| `locale` | BCP 47 | Per-trigger locale override |
| `slackWebhookUrl` | URL | Per-subscriber Slack override |
| `discordWebhookUrl` | URL | Per-subscriber Discord override |
| `teamsWebhookUrl` | URL | Per-subscriber Teams override |

## Schedules

```ts
await nexus.schedules.create({
  name: 'Monday digest',
  workflowId: 'workflow-uuid',
  cronExpression: '0 9 * * 1',
  timezone: 'UTC',
  recipients: [{ externalId: 'user_1' }],
  data: {},
});

await nexus.schedules.list({ page: 1, limit: 20 });
await nexus.schedules.update(id, { active: false });
await nexus.schedules.delete(id);
```

## Resilience

- **Circuit breaker** — opens after 5 failures, auto-recovers after 60s
- **Retry buffer** — failed requests queued in memory (max 500)
- **`flushRetries()`** — replay buffered triggers when platform is healthy

```ts
const nexus = new NexusClient({
  secretKey: process.env.NEXUS_SECRET_KEY!,
  events: {
    onCircuitOpen: () => console.warn('Nexus circuit open'),
    onCircuitClose: () => nexus.workflows.flushRetries(),
    onBufferOverflow: (p) => console.error('Dropped:', p.workflowName),
  },
});
```

## Response shape

```ts
type NexusResult<T> = {
  ok: boolean;
  status: number;
  data?: T;
  error?: string;
  queued?: boolean; // true if buffered locally
};
```

Successful triggers return **HTTP 202** with `{ accepted: true, results: [...] }`.
