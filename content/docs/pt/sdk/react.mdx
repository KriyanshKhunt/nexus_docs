## NexusProvider

Envolva seu aplicativo (ou layout autenticado):

```tsx title="app/layout.tsx"
import { NexusProvider } from "@nexus-signal/react";

export function AppShell({ user, hmac }: { user: User; hmac: string }) {
  return (
    <NexusProvider
      publicKey={process.env.NEXT_PUBLIC_NEXUS_PUBLIC_KEY!}
      subscriberId={user.id}
      hmacSignature={hmac}
      backendUrl="https://api.nexussignal.dev"
    >
      {children}
    </NexusProvider>
  );
}
```

## Sino de notificação

```tsx title="components/header.tsx"
import { NotificationCenterBell } from "@nexus-signal/react";

<header>
  <NotificationCenterBell position="bottom-end" variant="ghost" />
</header>;
```

- Atualizações em tempo real por WebSockets com selo de não lidas
- Ações para marcar como lida e arquivar
- Sincronização de leitura entre múltiplos dispositivos

## Inbox de notificações

Feed de página inteira com agrupamento por datas e rolagem infinita:

```tsx title="pages/notifications.tsx"
import { NotificationInbox } from "@nexus-signal/react";

<NotificationInbox />;
```

Utiliza `GET /v1/sdk/notifications` com paginação (`hasMore`, `loadMore`).

## Matriz de preferências

```tsx title="pages/settings.tsx"
import { SubscriberPreferenceMatrix } from "@nexus-signal/react";

<SubscriberPreferenceMatrix variant="borderless" groupByTopic />;
```

Renderiza eventos de fluxo de trabalho ou **tópicos** com caixas de seleção de canais. As alterações persistem através da API de preferências do SDK.

## Hook useNotifications

```tsx title="components/notification-list.tsx"
import { useNotifications } from "@nexus-signal/react";

const { items, unread, status, loadMore, hasMore, markRead, markAllRead } =
  useNotifications();
```

## Temas

Os componentes usam variáveis CSS (`--nexus-theme-*`) compatíveis com Tailwind. Sobrescreva no seu CSS global para combinar com a sua marca.

## Segurança

<Callout type="error">
  Gere o `hmacSignature` no seu servidor para cada sessão. Nunca coloque a
  `secretKey` no código do frontend.
</Callout>
