O Nexus Signal integra-se com **suas contas de provedores existentes**. Configure as chaves uma única vez na central de **Integrations (Integrações)** do painel de controle.

## Camadas de integração

| Camada | Exemplos |
|-------|----------|
| E-mail | SendGrid, Resend, Amazon SES, Postmark, Mailgun |
| SMS | Twilio, Vonage, Plivo, Sinch |
| WhatsApp | Meta Cloud API, Twilio WhatsApp, Infobip |
| Push | Firebase FCM, Apple APNs |
| Mensageria | Slack, Discord, Microsoft Teams |
| Webhook | Endpoints HTTP personalizados |
| IA | OpenAI, Anthropic (geração de conteúdo) |

## Configuração de pesos

Quando existem múltiplos provedores para uma mesma camada, defina os pesos de tráfego (a soma dos pesos deve totalizar 100):

```http
PUT /v1/integrations/weights/email
{ "sendgrid": 70, "resend": 30 }
```

Camadas disponíveis: `email`, `sms`, `whatsapp`, `push`, `webhook`, `slack`, `discord`, `ms_teams`

<Callout type="info">
Consulte [BYOP](/docs/platform/concepts/byop) para entender por que o Nexus nunca cobra taxas adicionais sobre os custos dos provedores.
</Callout>
